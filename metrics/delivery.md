# Metrics · Delivery（交付自动化）

> 对应引擎：Service Engine + AI Operating System。最大隐藏风险（BOS 指令）：**人工交付比例过高会拖垮单位经济**。
> 用 **Delivery Automation Ratio** 量化"机器做多少、人做多少"。

## 1. Delivery Automation Ratio（核心铁律指标）

| 环节 | 定义 | 目标自动化率 | 起点(2026-07-23) | 现状说明 |
|---|---|---|---|---|
| 资料收集 | 商家信息/GBP 抓取/文案初稿自动 | **90%** | ~70% | OSM 抓取 + AI Intake 草稿已半自动；邮箱仍需人工补 |
| 网站生成 | `generate.mjs` 一键出站 | **95%** | ~95% | 模板引擎成熟，单命令构建 |
| SEO | sitemap/JSON-LD/og 自动注入 | **80%** | ~80% | gen-seo.mjs 已自动 |
| 修改 | 客户反馈→改站 | **50%** | ~20% | 多靠人工改 JSON；待 AI 改稿闭环 |

- **综合自动化率目标**：加权 ≈ **(90+95+80+50)/4 ≈ 79%**。
- 起点约 **(70+95+80+20)/4 ≈ 66%** → 验证期重点把"修改"从 20% 拉到 50%。

## 2. 交付时长（Speed to Value）

| 环节 | 目标 | 起点 |
|---|---|---|
| 首触→首样（3 条评价草稿） | < 24h | 已做到（样例预生成） |
| 成交→建站上线 | < 7 天 | 模板站 < 1 天；定制内容待填 |
| 成交→Care/订阅激活 | < 48h | widget 自包含 snippet 可即时 |

## 3. 人工时（Unit Economics）

- **目标**：单客户全生命周期人工时 < 6h（建站 < 4h + 首月维护 < 2h）。
- 起点估算：建站人工 ~3h（生成 0.5h + 内容打磨 2.5h）+ 维护 ~1h/月。
- 监控：每成交一单在 `customer-system/<slug>/` 记 `delivery` 实际人工时，回填本表。

## 4. 质量门（不自动化妥协）

- 真实数据零失真：自动生成禁止虚构评分/评价（铁律 B8）。
- 每站上线前 `validate-sites.mjs` EXIT=0（残缺站阻断部署）。
- 修改自动化率上去的同时，首屏/文案仍过"去 AI 味 + 差异化"自检。

## 5. 周报字段

```
综合自动化率 / 资料% / 生成% / SEO% / 修改% / 本周人工时 / 超时交付数
```
