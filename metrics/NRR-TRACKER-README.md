# NRR & 单位经济追踪表 · 使用说明

> 工具：`metrics/nrr-tracker.html`（浏览器打开，离线可用，localStorage 持久化 + CSV 导入导出）
> 配套模型：`metrics/pure-subscription-model.md` v2（含保守 Churn / NRR）——本工具是其**运营层伴侣**，只做计算与记录，不另立口径。
> 状态：v1 · 2026-07-24 由 #433 搭建。内部运营资产，**不部署**到 thefifthstar.site。

## 为什么有这个工具

#433 的诉求是「搭 NRR / 单位经济追踪表」。模型文档已定好口径（四档、保守留存曲线、LTV 区间、NRR 目标），但缺一个**可每月回填、自动算 NRR/GRR/LTV、并和模型目标实时对比**的操作界面。本工具填这个缺口。

## 三个模块

### A. 单位经济模型（v2 口径）
- 四档月/年费取自 `docs/pricing.md` v2（年付为 #431 校正的折扣价 £390/£790/£1,490）。
- 假设可改（月流失率、毛利率、规划窗口月数、CAC），或一键「载入 v2 保守基准」。
- **Cohort LTV** = Σ(月=1..窗口) [ 留存(t)/100 × 月费 × 毛利率 ]，用模型文档的保守留存曲线 100→70→50→40 加权（非 £79×18）。
- **稳态 LTV** = 月费×毛利率÷月流失率（假设永久留存，作上限参考）。
- 验证（默认假设下）：Growth Partner Cohort LTV ≈ **£889**（落在 £800–1,000 区间）✓；年化流失 ≈49%（≈模型 50%）✓；LTV:CAC ≥10:1（冷邮件 CAC≈0）✓。

### B. 月度 NRR / GRR 追踪（核心追踪表）
- 每行 = 一个自然月。填五项：月初 MRR、新增、扩展、收缩、流失（收缩/流失带负号）。
- 自动算：月底 MRR、GRR =（月初+收缩+流失）÷月初、NRR =（月初+扩展+收缩+流失）÷月初。
- 「顺延」按钮把上月月底 MRR 自动填入本月月初，避免手抄错。
- NRR 走势图：绿=达标(≥110%)、金=≥100%、红=<100%；目标线 110% / 100%。
- 点「载入示例」看 6 个月演示（NRR≈111%）。

### C. 客户台账（背景）
- 纯记录：商家、起始档、起始月、当前档/月费、状态、备注。**不自动算 NRR**（避免与月度表重复记账）。NRR 以 B 表为准。

### D. 目标仪表盘
- 实时对比模型文档口径：NRR≥110% / 月 churn≤5% / 年化 churn≤50% / Cohort LTV £800–1k / LTV:CAC≥10:1 / 升档率≥40%（n/a 待更多数据）。
- 标「达标 / 观察·n.a / 未达标」。

### E. 导入 / 导出
- 导出 CSV（假设+月度+台账）作备份；导入 CSV 跨设备迁移。
- localStorage 是日常存储，**换浏览器/清缓存会丢**——每月底导一份 CSV。

## 使用节奏（建议）
- 每月底填一行月度 NRR（从上月月底顺延）；有升档/流失当天记台账。
- 连续 2 月未填即视为节奏退化，下月补。
- 首个客户成交后，用真实 churn / 扩展数据**复盘重定**假设与 `docs/pricing.md`（定价复盘约定）。

## 口径铁律（与模型文档一致）
- 四档：Free £0 / Reputation £39 / Growth Partner £79 / Growth Plus £149；年付 £390/£790/£1,490（省 17%）。
- 保守留存：cohort 100 起点，M1=100 → M6=70 → M12=50 → M18=40（年化≈50%、月≈5.5%）。
- 保守 LTV **£800–1,000**（Cohort 加权，非 £79×18）。
- NRR **目标 ≥110%**（Reputation→Growth Partner→Growth Plus 扩档 + 低 churn > 流失减量）。
- 任何口径调整以 `metrics/` 下文档为准；本工具仅计算与记录。
