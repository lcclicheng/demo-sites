# Metrics · Retention（续费→留存）

> 对应引擎：Customer Engine。原系统最大缺口（只有线索没有客户生命周期）。
> 数据源：`customer-system/<slug>/` + `business-profile.customer`。

## 1. 续费 / Churn

| 指标 | 定义 | 目标 | 起点 |
|---|---|---|---|
| 续费率（订阅层） | 到期续费数 / 到期总数 | **≥ 85%** | 0（无客户） |
| 年护续费率（Care） | Care 到期续费比例 | **≥ 80%** | 0 |
| 月 Churn | 当月取消 / 活跃订阅 | **≤ 5%** | 0 |
| 建站复购 | 建站客户加购 Care/SEO 比例 | **≥ 40%** | 0 |

## 2. 客户成功（续费前提）

| 指标 | 目标 | 起点 |
|---|---|---|
| 评价回复覆盖 | 客户 Google 新评 48h 内回 ≥ 90% | 0（未激活） |
| 评分趋势 | 续费前 90 天 rating 不降 | — |
| NPS | **≥ 50** | 未测 |
| 主动触达 | 每客户季度 ≥ 1 次价值回访 | 0 |

## 3. 单位经济（LTV : CAC）

| 指标 | 公式 | 目标 | 起点 |
|---|---|---|---|
| CAC | 获客总花费 / 新客数 | < £50 | 近乎 £0（冷邮件零广告） |
| LTV（订阅） | (ARPU × 毛利%) / Churn | **≥ £500** | 0 |
| LTV（建站+Care） | £590 + £149×12×0.8 | **≥ £2,000** | 0 |
| **LTV : CAC** | LTV / CAC | **≥ 10:1** | — |

> 冷邮件获客 CAC 极低（近乎仅时间成本），故 LTV:CAC 天花板很高——这是 FifthStar 单位经济的核心优势，须在验证期用真实续费数据证实。

## 4. 流失预警（Churn Risk）

- 信号：连续 30 天无新评可回 / rating 下降 / 未开账单 / 无回复。
- 触发：在 `customer-system/<slug>/success.md` 标 `at_risk`，Customer Engine 启动挽回序列。

## 5. 周报字段

```
活跃客户数 / 本月续费数 / 本月 Churn / 续费率 / NPS / LTV:CAC / at_risk 数
```
