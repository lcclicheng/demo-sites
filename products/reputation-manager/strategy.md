## 0. 为什么是这个方向（零客户前提下的最优解）

你现在的约束：**没有客户群、预算低、单人/极小团队、但已有一台 UK 建站+冷外联机器。**

80 个案例给出的硬规律，逐条对上这个约束：

| 案例规律（80 例实证） | 本方向如何满足 |
|---|---|
| 订阅制是命脉（月收入中位 $30K 全靠 recurring） | AI 声誉管家 = £X/月订阅，比你现在 £390/年 Care 厚得多 |
| 窄场景 > 泛化 | 只做"英国本地商户的 Google 评价/声誉"，不做通用 AI |
| 起步成本中位 $500、59% 在 $500 内 | MVP 只需 API 费 + 你已有的建站引擎，近乎零启动 |
| 39% 在 30 天内变现 | 冷外联 + 免费样例钩子，几周内可拿到第一个付费 |
| 增长靠口碑(61%)+SEO(39%)，付费广告仅 1% | 完全复用你的冷外联框架 + 内容站，0 买量 |
| 对标 Ian Kirby「Automated review generation」$300K/mo | 同赛道已被验证能做到头部 |

**关键：为什么不是"直接冷卖建站"？** 因为冷卖一个 £590 的网站，决策重、周期长、对方要换现有站、信任门槛高。而"帮你把最新 3 条 Google 差评的回复稿写好，免费发你看"——这是**即时、零风险、痛点当下**的钩子，回复率远高于卖网站。先用小订阅建立信任和现金流，**再把建站作为 upsell**。这就是"楔子进门 → 阶梯上升"。

## 1. 产品定义

**产品名（待定，先用代号）**：ReplyLocal / 「声誉管家」

**做什么**：给英国本地实体商户（餐厅、咖啡馆、美发/美甲、牙医、健身房、汽修…）提供 AI 驱动的 Google 评价与声誉管理：
1. **AI 回评**：自动为每条新 Google 评价生成得体、带品牌语气、符合英式表达的回复稿（好评致谢、差评安抚+引导线下解决）。
2. **声誉监测**：新评价即时提醒；评分趋势 + 关键词情绪周报（"顾客最常夸/最常抱怨什么"）。
3. **主动求评**：给到店顾客发求评链接/二维码，把满意顾客沉淀成 Google 好评。
4. **（进阶）本地 SEO 体检**：Google 商家资料完整度、竞品评分对比——这一步天然引出"你该有个更好的网站"。

**为什么是"评价/声誉"而不是"网站聊天机器人"**：聊天机器人要求对方先有网站；评价管理**不需要对方有网站**就能立刻交付价值 → 冷启动门槛最低。

## 2. 楔子 → 建站 的上升阶梯（这是整个模式的核心）

```
免费 AI 评价回复样例（冷外联钩子）
        │  回复率高、零风险、痛点当下
        ▼
£29–£79/月  AI 声誉管家订阅（recurring 现金流 + 建立信任）
        │  用了 1–2 个月、看到评分/客流改善、信任已建立
        ▼
£590 高端建站 + 首月不限改（你现有 A 档，此时是顺理成章的 upsell）
        │
        ▼
£390/年 Annual Care + 网站内嵌 AI 接待助手（把 AI 订阅并入，客单价再抬）
```

- **对你的意义**：不再需要冷卖高价网站。声誉管家帮你**低门槛拿下客户 + 每月现金流**，网站变成"已信任客户的自然升级"，转化率和客单价都更高。
- **对客户的意义**：先花小钱解决当下痛点，验证你靠谱，再放心把网站交给你。

## 4. MVP（低成本、快验证）

**第一版故意做"半自动"，不追求全自动化，先验证付费意愿（案例规律：先验证再投入）。**

- **技术栈**：沿用 Vite+React+Tailwind 落地页 + 一个轻后台；AI 用 OpenAI/Anthropic API 生成回复稿；评价数据初期**手动/半自动**拉取（Google Business Profile API 需商户授权，MVP 阶段可先让商户截图/授权只读，或用 SerpAPI 抓公开评价做样例）。
- **交付形态（MVP）**：每周给客户一份"AI 回复稿包"（Google Doc / 邮件）+ 一页声誉周报。客户满意 → 再投入做自助仪表盘和 GBP 自动同步。
- **成本**：≈ £0–£400 起步（API 费 + 域名；建站引擎已有）。完美落在案例中位 $500 起步成本内。
- **合规先行**：GDPR（处理顾客评价数据要有依据）、Google 商家资料条款（回复由商户账号发出，你是代运营工具）、AI 透明度（回复稿可标注"AI 辅助、人工确认"）。落地前把这三点在服务条款里写清。

## 5. 冷启动获客（零客户 → 前 10 个付费）

**复用你的 6 段冷外联框架，把"免费建站预览"钩子换成"免费评价回复样例"钩子：**

**新版 6 段（首触邮件/GMB 留言）**：
1. **来由（真实）**：我在 Google 上看到你家 [店名] 有 [X] 条评价、评分 [4.3]（真实数据，证明我做了功课）。
2. **痛点定位**：大多数本地商户没时间逐条回复评价，但回评能提升 Google 排名与新客信任。
3. **已代做样例（降门槛）**：我用 AI 帮你把最近 3 条评价（含那条差评）的回复稿写好了，附在下面，你可直接复制粘贴到 Google——**完全免费**。
4. **所得 bullet**：得体英式语气 / 差评转危为机 / 每周新评自动搞定 / 省你时间。
5. **市场对比**：竞品 [同城对手] 评分 4.6 且条条回复，正在被 Google 优先展示——差距可快速拉平。
6. **付费不伤钩子**：如果想每周自动收到回复稿 + 声誉周报，有个 £29/月 的小方案，无压力；样例先用着。

**分发**：
- 有邮箱的商户 → 邮件 API（`send-outreach.mjs`，带退订头，合规）。
- 无官网/无邮箱 → GMB 留言 / FB 私信（你本人账号手动粘贴，AI 不代发，避免封号）。
- 目标池：先锁 **1 个垂直 + 1 个城市**（建议：曼城/伯明翰的独立餐厅或美发店——评价量大、痛点强、决策快）。窄，才能出口碑和案例。

**节奏**：沿用 4 / 10 / 14 天跟进。每天发 10–20 家，一周 100 家，按案例经验转化率跑几周就能出前 3–5 个付费。

## 8. 风险与红线

- **Google 条款**：评价回复必须由**商户自己的 GBP 账号**发出，你是"代运营/工具"，不冒充商户批量自动化操作，避免封号（与外联 skill 的 GMB 红线一致）。
- **GDPR**：处理顾客评价/联系数据要有合法依据，服务条款写清；求评链接需顾客自愿。
- **AI 透明度**：回复稿可注明"AI 辅助、商户确认后发布"，避免"假装真人"争议。
- **不虚构**：外联里的评分/评价数据必须真实（你 skill 里本就有此红线）；免费样例要真能用。
- **窄聚焦**：别一上来铺多个行业多个城市——案例规律是"窄场景才出口碑"，先打穿一个。
