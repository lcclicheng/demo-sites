## 货币与税务

- 报价货币：**英镑（£ GBP）**，面向英国小商家。
- 价格**不含 VAT**（英国 VAT 注册阈值 £85,000/年；未达阈值不强制收 VAT，但报价标注 ex VAT 是专业惯例）。如未来注册 VAT 另议。
- 建站费为一次性；年度支持按年续费。

## 套餐（A/B 续费档 + C 买断档）

> 结构说明：A 档 = **建站 + 首月内不限次数修改**（由你直接改，不交付客户自助后台——省去为每个客户配置 OAuth、培训后台、长期维护的隐性成本）；「年费支持」单独为 B 档，首月后任何改动走 B；**C 档 = 一次性「完整买断交付」**，把站点源码 + 数据 + 自助编辑后台（Decap CMS，指向客户自己的 GitHub 仓库，一次性 setup、运维归客户）整体移交，客户此后自行改内容、自行运维，零你的后续维护。本结构为**进攻型**，目的为多接单、降低后续维护负担；首个客户交付后复盘重定。

### A — 建站 + 首月无忧改版（一次性）£590

适合：想立刻拥有一个专业、可上线、零托管成本的站点，且希望上线初期能随意调整的小商家。把「每个内容都要自己改」变成「首月内你帮我改到满意」——客户零后台学习成本，你也无需维护自助后台。

包含：

- 1 个高端静态站点（基于 8 套行业模板之一，Vite + React 18 + TS + Tailwind）
- 部署到 GitHub Pages（零服务器成本），含 SEO 基础（og: / sitemap / robots）、隐私政策页自动注入
- 交付标准化包（`docs/delivery-handover.md`）：欢迎邮件、合规交付清单签字版、站点维护手册（一页纸）、支持条款、发票
- **上手卡（一页纸）**：客户如何查看站点、如何向我提修改需求、如何续费域名（非 CMS 后台操作指南）
- **上线后 7 天**邮件 / 微信培训支持（见 `docs/workflow.md` §5.10 反馈循环）
- **首月内不限次数修改**：交付后 30 天内，任何文案 / 图片 / 菜单 / 营业时间调整，邮件告诉我即可，不限轮次、免费改到满意

> 差价值：取消 CMS 自助后台，改为「首月你代改 + 之后走年度呵护」——客户零后台学习成本，你也省去为每个客户配 OAuth、培训、维护后台的隐性负担；且首月后改动自然导向 B 档续费，黏性更强。

### B — 年度呵护（年费）£390 / 年

在 A 基础上续费（按年）：

- 站点健康监控（UptimeRobot + GitHub Issues 看板，异常主动告警）
- **优先响应**：支持请求 48 小时内响应（具体 SLA 见支持条款）
- **免费小改动额度**：含合理频次的小改动（如改一段文案 / 加一项菜单 / 换一张图）；额度与频次在支持条款中明确
- 年度域名 / DNS 续费协助（若客户用自定义域名，见 `docs/custom-domain.md`）
- 模板小幅升级（如新增板块）享折扣

> **客户首年总支出 = £590 + £390 = £980**。对比市场 freelancer（一次性 £1,000–£3,000 + 维护 £300–£1,200/年），总价更低、你利润反而更好——静态站维护极轻，B 档近乎纯毛利。

> **首月后改动归口**：A 档的「首月无忧改版」结束后，任何内容修改走本 B 档（或单次付费改版），**不再提供客户自助后台**——所有改动由你经 GitHub Pages 重建上线，可控、可回滚。

### C — 完整买断交付（一次性）£1,200

适合：想要**完全拥有站点、自己掌控内容与运维**、不愿长期付年费的小商家 / 有内部技术资源者。把「建站 + 终身自助 + 源码所有权」一次性买断。

包含（在 A 档全部交付物基础上，额外移交）：

- **站点源码仓库**：整套工程（generate.mjs / 8 套模板 / 共享组件 / 脚本）转移到**客户自己的 GitHub 账号**（或新建其名下仓库），客户完全拥有、可自由 fork / 改模板 / 自部署。
- **数据所有权**：客户站点的 `examples/<slug>.json` 归客户，自行掌控全部内容。
- **自助编辑后台（Decap CMS）一次性配置**：指向客户自己的仓库 + 客户自己注册的 GitHub OAuth App（免费），之后客户在网页 `/admin` 改内容 → 自动 commit + 部署，**零你的后续维护**（区别于 v0.9.2.2 取消的「我们维护的后台」——此处运维全归客户）。
- **交接文档 + 短时交接培训**：源码结构说明、Decap 用法一页纸、如何自部署 / 自改域名。

> 差价值：客户拿到「源码 + 数据 + 自助后台」完整资产，终身自己管；你一次性收 £1,200（吃掉约 3 年 B 档年费价值 + 一次性配置工时），交付后**零维护负担、零年费管理**。对比市场 freelancer 买断（£1,000+ 起、多为套模板），本方案含定制设计语言 + 自助后台，竞争力强。

> **与 B 档关系**：C 档默认独立——客户自行运维、不强制买 B。若客户仍想要监控 / 优先响应，可**按需叠加 B**（可选），但 C 已能自助，B 转化率低，属增值而非默认。

## 声誉管家订阅 · ReplyLocal（冷启动楔子 + 月收入层）

### 上升阶梯（楔子 → 建站，核心模式）

```
免费评价样例  →  £29/月（信任 + 现金流）
        │  用 1–2 个月、看到评分/客流改善、信任已建立
        ▼
£590 高端建站（A 档，此时是顺理成章的 upsell）
        │
        ▼
£390/年 Annual Care + 网站内嵌 AI 接待助手（把订阅并入，客单价再抬）
```

客户先花小钱解决当下痛点，验证你靠谱，再放心把网站交给你——转化率与客单价都高于冷卖高价网站。

### 合规红线（与外联 skill 一致）

- **Google 条款**：评价回复必须由**商户自己的 GBP 账号**发出，你是代运营 / 工具，不冒充批量自动化操作，避免封号。
- **GDPR**：处理顾客评价数据要有合法依据，服务条款写清；求评链接需顾客自愿。
- **AI 透明度**：回复稿可注明「AI 辅助、商户确认后发布」，避免「假装真人」争议。
- **不虚构**：外联里引用的评分 / 评价必须真实（沿用现有红线）；免费样例要真能用。

## 分行业定价调整（按层次微调）

> 原则：保留上方两档（A £590 / B £390）为**基准锚点**；按行业「支付力 / 形象溢价」分 3 档，**仅做轻微调整**（幅度 ±£50 一次性 / ±£40 年费），不破坏进攻型定价。首个客户交付后随整体复盘重定。

### 三档映射（8 行业）

| 档位 | 行业（`template`） | A 一次性 | B 年费 | 首年合计(A+B) | C 买断一次性 |
|---|---|---|---|---|---|
| **高 Premium** | 律所 `law`、酒店 `hotel` | **£640** | **£430** | £1,070 | **£1,250** |
| **标准 Standard** | 餐饮 `restaurant`、沙龙 `salon`、甜品 `dessert`、咖啡 `coffee`、瑜伽 `yoga` | £590 | £390 | £980 | £1,200 |
| **经济 Value** | 家装 / 维修 `trades` | **£540** | **£350** | £890 | **£1,150** |

> C 档为一次性买断，与 A 同幅度 ±£50（Premium +£50 / Value -£50），**不叠加 B 年费**（客户选 C 即自行运维，不再买 Annual Care，除非按需单独叠加）。

### 分档逻辑
- **高 Premium**：律所客户预算高、对专业形象溢价接受度最高；精品酒店有市场预算、站点直接影响预订转化 → 上浮。
- **标准 Standard**：生活方式类小商家，形象敏感但预算居中、价格弹性中等 → 取基准。
- **经济 Value**：家装 / 维修多为 sole trader，极价格敏感；以略低入门价换量与口碑 → 下浮。

### 使用约定
- 报价时以本表对应行业的档位价为准；两档结构、包含项、交付包**完全一致**，仅金额按档微调。
- 单行业如需特殊偏移（如某高端餐厅想对标 Premium），可在本表基础上个案协商，并在交付包注明。
- 复盘重定（见文末约定）时一并审视分档幅度是否到位。
