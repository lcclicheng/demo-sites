# 合规交付清单（Compliance Delivery Checklist）

> 配套 `workflow.md` §7（业务合规与法律风险）使用。每接一个真实客户、每次上线前，**逐项核对并保留记录**，降低你的法律与声誉风险。
> 本文档是「交付前把关清单」，不替代专业法律意见。涉及具体合规判断时，提示客户自行咨询会计师 / 律师。

---

## 0. 风险边界（务必先读）

- 你**只做模板注入**（把客户提供的文字 / 地址 / 图片填进字段），**不替客户做法律判断**。
- 注册地址是否「ECCTA 合规的适当地址」、隐私条款是否真实合规，由**客户负责**；你仅在模板 / 交付清单中提示，不担保。
- GitHub Pages 站点**完全公开**：JSON 里的电话、地址等客户信息对外可见。交付前必须提醒客户此点。

---

## 1. 交付前必查（owner 把关，逐项打勾）

| # | 检查项 | 说明 / 操作 | ✅ |
|---|---|---|---|
| 1 | **隐私政策页已注入** | `inject-privacy.mjs` 已自动注入 Privacy 页；但内容为**占位模板**，需提醒客户补充真实条款 | ☐ |
| 2 | **GDPR / Cookie 同意** | 英国 GDPR 下需真实隐私政策 + Cookie 同意机制。**当前模板无 Cookie 同意弹窗**；若客户站用了分析 / 追踪工具（GA、Meta Pixel 等），必须补 Cookie 同意，否则违规 | ☐ |
| 3 | **注册地址合规（Ltd 客户）** | `registeredAddress` 须为 ECCTA 2024 合规的「适当地址」（能收官方信件），**非纯 PO Box**。住宅 / 会计师免费地址 / 虚拟办公室均可 | ☐ |
| 4 | **业务类型字段正确** | Ltd / Sole Trader 切换正确：Sole Trader 时 `registeredAddress` 应清空禁用（无注册办公室概念） | ☐ |
| 5 | **公开信息提醒** | 已明确告知客户：电话 / 地址 / 营业时间等会公开可见 | ☐ |
| 6 | **内容准确性** | 菜单、价格、营业时间、电话、邮箱、社交链接均经客户确认无误 | ☐ |
| 7 | **图片版权** | 客户上传的实景图，客户拥有版权或已获授权；占位图（Openverse）仅演示用，交付前须替换 | ☐ |
| 8 | **SEO / 社交卡片** | 已设 `pageTitle` / `tagline`（驱动 meta + OG）；交付后可在搜索引擎 / 社媒正常预览 | ☐ |

---

## 2. 上线后（部署完成）

- [ ] 硬刷新（Ctrl+F5）验证线上页面、图片、预览链接均正常
- [ ] 用社交卡片调试器（如 Facebook Sharing Debugger / Twitter Card Validator）验证 OG 图与描述
- [ ] 提交 `sitemap.xml` 到 Google Search Console / Bing Webmaster（提升收录）
- [ ] 如用自定义域名，见 [`custom-domain.md`](./custom-domain.md) 完成 DNS / SSL / 交接

---

## 3. 客户交接清单（发给客户存档）

- 线上地址：`https://lcclicheng.github.io/demo-sites/<slug>/`（或客户自有域名）
- 后续改内容方式：通过你（owner）改 JSON + 重建部署（见 `workflow.md` §5.9），客户不直接碰代码
- 域名 / DNS / SSL（若启用自定义域名）：账号、权限、续费日期一并交代
- 提醒：隐私政策与 Cookie 同意需客户自行完善真实条款
